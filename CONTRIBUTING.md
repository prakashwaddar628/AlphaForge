# Contributing to AlphaForge

Thank you for your interest in contributing to AlphaForge! We welcome contributions from the community to help make this platform better for everyone.

## How to Contribute

### 1. Reporting Bugs
- Use the GitHub Issues tracker to report bugs.
- Include detailed steps to reproduce the issue and your environment details.

### 2. Suggesting Enhancements
- Open an Issue with the prefix `[Feature Request]`.
- Describe why this feature would be useful and how it should work.

### 3. Pull Requests
- Fork the repository.
- Create a new branch for your feature or bugfix.
- Ensure your code follows the existing style and includes tests if applicable.
- Submit a PR with a clear description of the changes.

## Development Setup
Refer to the [README.md](./README.md) for installation and setup instructions.

## Coding Standards
- **Python**: Follow PEP 8 guidelines. Use docstrings for all functions and classes.
- **Frontend**: Use TypeScript and Tailwind CSS patterns. Functional components with hooks are preferred.

## Code of Conduct
Please be respectful and professional in all interactions within the community.
